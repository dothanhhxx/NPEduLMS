USER STORY: VIEW STUDENTS LIST
Card
As a teacher 
I want to view the list of students in my class 
So that I know who I am teaching.
Summary
Allows teachers to view the list of students enrolled in their class, including student information and learning status.
In-Scope:
Teachers can select a class to view the student list.
The system displays a list of students in the selected class.
Teachers can search students by name, student ID, or phone number.
Teachers can navigate between pages of the student list.
Out of scope
Teachers cannot edit student information.
Teachers cannot add new students.
Teachers cannot delete students from the system.
Teachers cannot view students from other classes they do not teach.
Trigger
Student opens the Student List page after login
Pre-condition
Teacher must be logged in.
Teacher must be assigned to at least one class.
Student data must exist in the system.
Business rule and error message
Teachers can only view students from classes they are assigned to.
Students are displayed in a table format.
Teachers can search students using name, student ID, or phone number.
Student status can be displayed (e.g., Studying, On hold).
No students found -> shows "Không có học sinh nào trong lớp."

USER STORY: UPLOAD LEARNING MATERIALS
Card
As a teacher, 
I want to upload learning materials 
So that students can access learning resources.
Summary
Allows teachers to upload learning materials (PDF, video, slides, documents) for their assigned courses.
In-Scope:
Uploading physical files (PDF, DOCX, PPTX, MP4).
Adding material titles.
Assigning the material to a specific course and lesson/module.
Out of scope
Bulk upload multiple files at once
Preview file before upload
File size auto compression
Integration with external storage (Google Drive, Dropbox)
Tracking whether a student has completely read/watched the material.
Trigger
Teacher clicks the "Upload Material" button from the Learning Materials page.
Pre-condition
Teacher must be logged in.
Teacher must be assigned to at least one class.
Student data must exist in the system.
Business rule and error message
Teacher can only upload materials to courses they are assigned to.
File size must not exceed 100MB.
Only allowed file formats can be uploaded.
Title and File cannot be empty. If empty, show: “Vui lòng nhập tên tài liệu và chọn tệp tải lên.”
If file format invalid → show:
 “Định dạng tệp không được hỗ trợ. Vui lòng chọn tệp PDF, Word, PowerPoint hoặc Video.”
If file exceeds size limit → show:
 “Dung lượng tệp vượt quá giới hạn cho phép (Tối đa 100MB).”
If upload successful → show:
 “Tài liệu tải lên thành công”

USER STORY: CREATE ASSIGNMENT
Card
As a teacher, 
I want to create assignments 
So that students can practice what they learned.
Summary
Allows teachers to create assignments for their assigned courses.
In-Scope:
Setting assignment title and detailed instructions.
Setting a start date and a deadline (due date).
Attaching reference materials or template files for the assignment.
Out of scope
Setting the maximum score for the assignment.
System sends notification to enrolled students after assignment creation
Automatic grading
Trigger
Teacher clicks the “Create Assignment” button from the Assignment Management page.
Pre-condition
Teacher must be logged in.
Teacher must be assigned to at least one class.
Course must exist in the system
Business rule and error message
Teacher can only create assignments for courses they are assigned to.
Assignment title cannot be empty.
Due date must be in the future and cannot be earlier than current date/time. If invalid, show: "Thời gian bắt đầu không hợp lệ."
Attachment file size must not exceed 100MB. If exceeded, show: “Dung lượng tệp đính kèm vượt quá 100MB.”
Title, Course/Lesson, Start Date, and Deadline cannot be empty. If empty, show: “Vui lòng điền đầy đủ các trường thông tin bắt buộc.”
If creation successful → show:
 “Tạo bài tập thành công.”

USER STORY: ASSIGN TEACHERS TO CLASSES
Card
As an admin, 
I want to assign teachers to classes
So that each class has an instructor.
Summary
Allows administrators to assign teachers to specific classes in the system.
In-Scope:
Admin can view list of classes
Admin can view list of teachers
Admin can assign one teacher to one class
Admin can view current teacher assigned to each class
Checking for scheduling conflicts for the selected teacher.
System updates teaching schedule automatically after assignment
Out of scope
Assign multiple teachers to one class (team teaching)
Automatic teacher-class matching
Trigger
Admin clicks the “Assign Teacher” button from Class Management page.
Pre-condition
Admin must be logged in
Teacher account must exist and be active
Class must exist in the system
Business rule and error message
One class can only have one assigned teacher.
A teacher can be assigned to multiple classes.
Admin cannot assign inactive teachers.
If no teacher is selected → show:
 “Vui lòng chọn một giáo viên chính cho lớp học”
If class already has a teacher → show confirmation message before reassigning.
If assignment successful → show:
 “Giáo viên này đã được phân công vào lớp học.”
The system must check the teacher's current schedule. If the teacher is already assigned to another class that takes place at the exact same time, show: “Giáo viên này đã có lịch dạy trùng vào thời gian của lớp học. Vui lòng chọn giáo viên khác.”

USER STORY: ENROLL STUDENT IN CLASS
Card
As an admin, 
I want to enroll students in classes 
So that they can attend the appropriate courses.
Summary
Allows administrators to assign students to specific classes in the system
In-Scope:
Admin can view list of students
Admin can view list of classes
Selecting a class and adding single or multiple students to it.
Admin can view list of students currently enrolled in a class
System updates student schedule automatically after enrollment
Searching for existing students by Name, Email, or Student ID.
Out of scope
Automatic class placement based on level
Payment processing
Trigger
Admin clicks the “Enroll Student” button from Class Management page.
Pre-condition
Admin must be logged in
Student account must exist and be active
Class must exist in the system
Class must not exceed maximum capacity
Business rule and error message
A student can enroll in multiple classes.
A class can contain multiple students.
Student cannot be enrolled twice in the same class.  If the student is already on the roster, show: “Học viên này đã có trong danh sách lớp.”
Enrollment is blocked if class reaches maximum capacity. If adding the selected student(s) exceeds this limit, show: “Lớp học đã đạt số lượng tối đa. Không thể thêm học viên.”
If enrollment successful → show:
 “Đã thêm học viên vào lớp thành công.”

USER STORY: SUBMIT HOMEWORK
Card
As a student, 
I want to submit homework through the system 
So that I can send my work to the teacher easily.
Summary
Allows students to upload and submit homework files before the assignment deadline.
In-Scope:
Upload and submit homework before deadline
View submission status (Submitted / Late / Not submitted)
System records submission date and time
Student can resubmit homework before deadline
Student can view submitted file
Out of scope
Automatic grading
Plagiarism detection
Group submission
File editing within system
Trigger
Student clicks the “Submit Homework” button from the Assignment Detail page.
Pre-condition
Student must be logged in
Student must be enrolled in the course
Assignment must exist
Assignment deadline must not be expired (for on-time submission)
Business rule and error message
Students can only submit assignments of enrolled courses.
Submission must be before deadline for on-time status.
If submission after deadline → status automatically marked as “Nộp muộn”.
File size must not exceed 100MB.
Only allowed file types (PDF, DOC, DOCX) can be uploaded.
If no file selected → show:
 “Vui lòng tải lên tệp.”
If file format invalid → show:
 “Tệp tải lên không đúng định dạng.”
If file exceeds size limit → show:
 “Dung lượng tệp vượt quá giới hạn cho phép (Tối đa 100MB).”
If submission successful → show:
 “Nộp bài tập thành công.”
